# 特赞项目深度文案（产品经理视角）

## 🤖 atypica - 商业研究多智能体系统

### 项目背景与痛点
传统商业研究存在三大核心痛点：
1. **信息过载**：面对海量行业报告、新闻资讯，分析师80%时间消耗在信息筛选上
2. **视角单一**：单个分析师难以同时从财务、战略、市场、技术等多维度切入
3. **知识断层**：过往研究经验难以系统化传承，每次项目从零开始

### 产品定位
**"AI 分析师团队"** —— 基于 Multi-Agent 架构的智能商业研究平台，模拟真实专业团队的协作模式，让 AI 代理人各司其职、协同输出洞察。

### Agent 架构设计

#### 1. 信息采集 Agent（Information Gatherer）
- **角色职责**：像初级分析师一样进行信息检索与筛选
- **工作流**：
  1. 接收研究主题 → 2. 构建检索策略（关键词+时间范围+信源权重）→ 3. 并行抓取公开数据（财报、新闻、研报）→ 4. 初步去重与质量打分
- **核心技术**：RAG + 网页抓取 + 数据质量评估模型
- **输出**：结构化信息库（带置信度标签）

#### 2. 领域专家 Agent（Domain Expert）
- **角色职责**：模拟不同专业视角的资深分析师
- **Agent 类型**：
  - **财务专家 Agent**：擅长财务建模、现金流分析、估值计算
  - **战略专家 Agent**：聚焦商业模式、竞争格局、增长路径
  - **技术专家 Agent**：评估技术壁垒、研发投入、专利布局
  - **市场专家 Agent**：分析用户画像、市场规模、获客成本
- **协作机制**：每个专家 Agent 独立分析信息库，生成视角专属的洞察报告

#### 3. 洞察综合 Agent（Insight Synthesizer）
- **角色职责**：像项目负责人一样整合多 Agent 输出
- **工作流**：
  1. 读取所有专家 Agent 的分析报告
  2. 识别观点冲突（如"财务乐观" vs "市场悲观"）
  3. 交叉验证数据支撑强度
  4. 生成带论证链的综合洞察
- **输出形式**：结构化研究报告（论点+论据+置信度+风险提示）

#### 4. 质量把关 Agent（Quality Controller）
- **角色职责**：模拟研究总监进行质量审核
- **检查维度**：
  - 逻辑一致性（论据是否支撑论点）
  - 数据真实性（引用源是否可靠）
  - 结论有效性（是否存在过度推断）
- **输出**：质量评分 + 修改建议

### 关键技术栈
- **Multi-Agent 协作框架**：实现 Agent 间消息传递、任务分发、结果汇总
- **知识图谱（Knowledge Graph）**：构建行业实体关系网络，支持关联分析
- **检索增强生成（RAG）**：确保分析基于真实数据源，避免幻觉
- **思维链提示工程**：引导 Agent 进行分步推理，提升复杂问题拆解能力

### 业务价值
- **效率提升**：从 5 天缩短到 4 小时完成一份行业深度研究
- **视角全面**：自动覆盖 4+ 专业维度，避免认知盲区
- **可追溯性**：每个结论均可追溯到原始数据源与推理逻辑
- **持续学习**：研究过程沉淀为知识资产，越用越聪明

## 🎤 pitchlab - AI 驱动的表达能力训练产品

### 项目背景与痛点
表达能力是职场核心竞争力，但传统训练方式存在：
1. **缺乏客观反馈**：依赖他人主观评价，难以量化进步轨迹
2. **练习场景有限**：缺少真实的演讲/汇报/面试模拟环境
3. **迭代成本高**：录制→回看→自我检视→修正，循环周期长
4. **针对性弱**：泛泛的建议（"要自信"）无法指向具体改进点

### 产品定位
**"AI 表达教练"** —— 基于多模态分析的实时表达反馈系统，让每一次开口都成为数据驱动的刻意练习。

### 核心功能架构

#### 1. 实时语音分析（Speech Analysis）
- **分析维度**：
  - **语速控制**：实时计算每分钟字数，识别过快/过慢段落
  - **填充词检测**：自动标记"然后"、"就是"、"那个"等口头禅
  - **停顿分析**：识别停顿位置是否合理（如句号处停顿 vs. 突然卡壳）
  - **音量波动**：检测是否出现声音变小（可能是紧张信号）
  - **语调变化**：分析抑扬顿挫，识别单调乏味的风险
- **技术实现**：
  - 语音识别（ASR）转写文本
  - NLP 分析文本特征（句长、词汇丰富度、逻辑连接词）
  - 声学特征提取（音高、能量、节奏）

#### 2. 非语言信号捕捉（Non-verbal Cues）
- **视觉分析**（需开启摄像头）：
  - **眼神接触**：计算视线与镜头的接触时长比例
  - **面部表情**：识别自然微笑 vs. 强颜欢笑、紧张僵硬
  - **肢体语言**：检测手部动作频率与幅度（如是否过度晃动）
  - **姿态稳定性**：识别身体晃动、重心转移等紧张表现
- **技术实现**：
  - 人脸关键点检测（Face Landmark Detection）
  - 表情识别（Emotion Recognition）
  - 姿态估计（Pose Estimation）

#### 3. 内容逻辑评估（Content Logic）
- **结构化分析**：
  - **开场吸引力**：检测是否使用钩子（故事、数据、提问）
  - **逻辑框架**：识别是否使用"总-分-总"、"问题-方案-论证"等结构
  - **证据支撑**：标记论点是否伴随具体案例/数据
  - **结尾力量**：检测是否有总结/呼吁/金句收尾
- **技术实现**：
  - 文本分类（判断段落功能：开场/论证/总结）
  - 论证链识别（论点→论据→结论的连贯性）
  - 修辞手法检测（排比、类比、对比等）

#### 4. 实时反馈机制（Real-time Feedback）
- **轻度干扰模式**：
  - 语音提示：轻柔的 AI 音提醒"语速过快"、"出现了3次'然后'"
  - 视觉提示：屏幕边缘浮现颜色标记（绿色=节奏好，黄色=注意，红色=需调整）
- **深度分析模式**：
  - 演讲结束后生成完整报告（60秒可快速浏览）
  - 时间轴标注关键节点（点击即跳转对应视频片段）
  - AI 生成针对性练习建议（如"停顿训练：每句话后停2秒"）

#### 5. 场景化训练库（Scenario-based Practice）
- **预设场景**：
  - 求职面试模拟（AI 扮面官提问，用户作答后双方都获反馈）
  - 产品汇报演练（上传 PPT，AI 模拟听众提问并评估回答质量）
  - 即兴演讲挑战（随机主题 + 限时准备 + 实时分析）
- **自定义模式**：用户上传自己的演讲视频，AI 进行赛后分析

### 技术栈
- **语音处理**：ASR（语音识别）+ TTS（语音合成）+ 音频特征提取
- **自然语言处理**：文本分析 + 情感识别 + 论证链提取
- **计算机视觉**：人脸/姿态检测 + 表情识别 + 视线追踪
- **多模态融合**：将语音、文本、视觉信号综合评估，避免单一维度误判

### 产品亮点
1. **隐私优先**：所有分析本地化处理，不上传原始视频
2. **进步可视化**：自动生成"表达能力成长曲线"，量化改进效果
3. **游戏化机制**：解锁成就徽章（如"流畅表达者"、"停顿大师"），提升练习动力
4. **社交对比**：匿名展示同类用户数据（如"你的语速超过了78%的用户"）

### 业务价值
- **To C**：帮助职场人系统提升表达力，建立个人品牌
- **To B**：企业培训场景，降低新员工汇报能力培养成本
- **数据资产**：沉淀"高质量表达"的多模态特征库，为未来更多应用（如虚拟人演讲）铺路

## 总结：两款产品的 Agent 思维共性

1. **角色分工明确**：每个 Agent 都有清晰的"人设"与职责边界
2. **协作而非单点**：通过 Agent 间交互产生 1+1>2 的效果
3. **闭环迭代**：输出结果反馈给系统，优化下一次 Agent 决策
4. **可解释性**：用户能看到 Agent 的思考过程，而非黑盒结果

这正是产品经理设计 AI 原生产品的核心能力：**把复杂任务拆解为可协作的智能单元，让系统像团队一样工作**。
